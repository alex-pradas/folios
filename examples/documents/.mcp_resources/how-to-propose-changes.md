# How to Propose Document Changes

Follow this workflow when proposing changes to an existing document.

## 1. Review the current version

Use `get_document_content` to read the latest version and understand the current state.

## 2. Identify what to change

Describe the change clearly: what section is affected, what the current text says, and what it should say instead.

## 3. Draft the change

Write the updated section in full, preserving the document's existing style and formatting conventions.

## 4. Provide context

Explain why the change is needed — reference requirements, standards, or review feedback as appropriate.
